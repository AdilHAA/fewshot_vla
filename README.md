# Hypernetwork-LoRA для SmolVLA

Гиперсеть, которая на каждом forward'е генерирует LoRA-адаптеры для замороженной
SmolVLA по текстовой инструкции (и, опционально, по кадру). Сама политика не
обучается — учится только гиперсеть, обычным flow-matching лоссом.

## Код

`hyper_lora/hypernetwork.py` — гиперсеть: кодирует инструкцию (трансформер или
mean-pool MLP) и выдаёт пары `(W_down, W_up)` для каждого MLP-слоя языковой части
VLM. Опционально на вход добавляются зрительные токены — собственные image-фичи VLM
и/или внешний замороженный DINOv2. Там же `StaticLoRABank` — безусловный
multi-task LoRA-бейзлайн (`static_lora=true`): те же точки врезки, но один
статичный набор адаптеров без кондишенинга.

`hyper_lora/dynamic_lora.py` — обёртка над `nn.Linear`: подставляет сгенерированные
LoRA-веса перед forward'ом и убирает после, так что они не являются параметрами модуля.

`hyper_lora/modeling_hyper_lora_smolvla.py` — связывает всё в lerobot-политику:
грузит базовый чекпоинт, патчит MLP-линейки VLM на `DynamicLoRALinear`, замораживает
всё кроме гиперсети, на каждом шаге считает LoRA из инструкции/кадра и инжектит.
Конфиг и переключатели зрительных веток — в `configuration_hyper_lora_smolvla.py`,
сопоставление полей с базовым чекпоинтом — в `base_config.py`.

`libero_pro/` поднимает сьюты LIBERO-Pro поверх родного рантайма LIBERO:
`register.py` раскладывает bddl/init и регистрирует сьюты, `objects.py` синтезирует
перекрашенные и увеличенные объекты для object-оси из базовых ассетов (LIBERO-Pro
поставляет только сцены, не меши).
